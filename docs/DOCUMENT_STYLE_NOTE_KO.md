# 문서 문체 점검 기록

이번 GitHub 문서는 과장된 칭찬, 번역체, 장황한 일반론을 줄이고 프로젝트 사실 중심으로 작성했습니다.

적용한 기준:

- 코드 블록을 남발하지 않음
- 굵게 표시하는 별표 표기를 쓰지 않음
- 파일명 설명은 문장 안에서 자연스럽게 표기
- 모델, 데이터, 실행 흐름, 결과 폴더만 설명
- 실제 데이터와 성능 수치가 없을 때는 그 상태를 그대로 밝힘
- 소스 파일 내용은 바꾸지 않음

업로드 시 README.md와 docs 폴더를 같이 올리면 GitHub 첫 화면과 문서 탭에서 포트폴리오 설명을 바로 확인할 수 있습니다.
