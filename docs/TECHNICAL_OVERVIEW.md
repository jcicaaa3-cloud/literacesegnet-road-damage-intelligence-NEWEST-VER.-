# Technical overview

This repository contains a road-damage segmentation pipeline with two separated model paths:

- **LiteRaceSegNet**: proposed lightweight CNN model
- **SegFormer-B3**: Transformer baseline for comparison

The repository is structured for capstone review and employment portfolio use. The focus is not only segmentation accuracy, but also deployment cost.

## Research question

> Can a custom lightweight CNN keep usable road-damage mask quality while reducing model cost and inference latency compared with a Transformer baseline under CPU and GPU conditions?

## LiteRaceSegNet

LiteRaceSegNet is implemented under `seg/core/lightweight_race.py` and trained through `seg/train_literace.py`.

The model uses:

- lightweight CNN stem and downsampling blocks,
- detail branch for local spatial information,
- LiteASPP-style context branch,
- boundary auxiliary output,
- boundary-guided fusion,
- segmentation output head.

## SegFormer-B3 baseline

SegFormer-B3 is kept under `seg/transformer_b3/`. It is a comparison path, not the proposed model.

This separation keeps the portfolio claim clean:

- LiteRaceSegNet checkpoint path is separate.
- SegFormer-B3 checkpoint path is separate.
- Inference outputs are saved separately.
- LLM/service explanation uses LiteRaceSegNet service summary only.

## Dual-device evaluation

The comparison script supports:

- `--device cpu`: no-GPU deployment evidence
- `--device cuda`: AWS/CUDA acceleration evidence

CPU and GPU measurements answer different questions. Compare models within the same device condition.

Exported fields include:

- parameter count
- estimated FP32 model size
- latency mean/std/min/max
- throughput FPS
- CUDA peak memory
- mIoU
- Damage IoU
- Pixel Accuracy
